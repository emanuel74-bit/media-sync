## Phase 5 ‚Äî Design Pattern Enforcement _(depends on Phase 4)_

### 5.1 Strategy: Injectable `SyncWorkflow[]` token in `SyncOrchestratorService`

**Problem:** `SyncOrchestratorService.execute()` hard-codes the 3 workflow implementations in a fixed array. Adding a new workflow requires modifying the orchestrator ‚Äî violates Open/Closed Principle.

**Solution:** NestJS multi-provider token pattern.

In `backend/src/sync/sync.module.ts`, define:

```ts
export const SYNC_WORKFLOWS = "SYNC_WORKFLOWS";
```

Register each workflow:

```ts
{ provide: SYNC_WORKFLOWS, useClass: IngestStreamSynchronizer, multi: true },
{ provide: SYNC_WORKFLOWS, useClass: StreamReconcileService, multi: true },
{ provide: SYNC_WORKFLOWS, useClass: StreamStalenessService, multi: true },
```

In `sync-orchestrator.service.ts`:

```ts
constructor(
    @Inject(SYNC_WORKFLOWS) private readonly workflows: SyncWorkflow[],
    private readonly queryAggregator: SyncQueryAggregatorService,
    private readonly scheduledWork: SequentialStreamTaskRunner,
) {}

async execute(context: SyncContext): Promise<void> {
    for (const workflow of this.workflows) {
        await workflow.execute(context);
    }
    this.events.emit(SystemEventNames.SYNC_TICK, { ... });
}
```

Adding a new `SyncWorkflow` now only requires registering a new provider token ‚Äî zero orchestrator changes.

### 5.2 Polymorphism: Replace `parseTrack()` switch in `StreamTrackParser`

**Problem:** `stream-track-parser.ts` has a `switch (track.type)` statement ‚Äî conditional branching that should be replaced with polymorphism.

**Create interface `backend/src/stream-inspection/services/parsing/track-parser.interface.ts`:**

```ts
export interface TrackParser {
    canParse(type: string): boolean;
    parse(track: V3TrackItem): StreamTrack;
}
```

**Create individual parsers in `backend/src/stream-inspection/services/parsing/parsers/`:**

- `video-track-parser.ts` ‚Äî `VideoTrackParser implements TrackParser`
- `audio-track-parser.ts` ‚Äî `AudioTrackParser implements TrackParser`
- `data-track-parser.ts` ‚Äî `DataTrackParser implements TrackParser`
- `subtitle-track-parser.ts` ‚Äî `SubtitleTrackParser implements TrackParser`

**Update `stream-track-parser.ts`:**

```ts
const TRACK_PARSERS: TrackParser[] = [
    new VideoTrackParser(),
    new AudioTrackParser(),
    new DataTrackParser(),
    new SubtitleTrackParser(),
];

export function parseTracksFromPathItem(item: V3PathItem): StreamTrack[] {
    return (item.tracks ?? [])
        .map(
            (track) =>
                TRACK_PARSERS.find((p) => p.canParse(track.type))?.parse(
                    track,
                ) ?? null,
        )
        .filter((t): t is StreamTrack => t !== null);
}
```

### 5.3 Factory + Caching: `MediaMtxClientFactory` in registry

**Problem:** `MediaMtxClientRegistry.getIngestClientsFromPods()` creates new `MediaMtxClient` instances on every call ‚Äî no caching.

**Create `backend/src/infrastructure/media-mtx/factory/media-mtx-client-factory.ts`:**

```ts
export class MediaMtxClientFactory {
    private readonly cache = new Map<string, MediaMtxClient>();

    getOrCreate(baseUrl: string): MediaMtxClient {
        if (!this.cache.has(baseUrl)) {
            this.cache.set(baseUrl, new MediaMtxClient(baseUrl));
        }
        return this.cache.get(baseUrl)!;
    }
}
```

Update `MediaMtxClientRegistry` to use `MediaMtxClientFactory` for all client creation (both static + dynamic from pod list).

## Phase 7 ‚Äî Query Consolidation in Sync _(depends on Phase 4)_

**Problem:** `SyncOrchestratorService` calls `buildContext()` then `withAllStreams()` as a second sequential async step, when the all-streams DB query could run in parallel with the MediaMTX queries.

**Update `SyncQueryAggregatorService.buildContext()`:**

```ts
async buildContext(): Promise<SyncContext> {
    const [ingestList, clusterList, podIds, allStreams] = await Promise.all([
        this.listIngestStreams(),
        this.listClusterStreams(),
        this.podQuery.listActivePodIds(PodRole.CLUSTER),
        this.streamQuery.findAll(),
    ]);
    return {
        ingestList,
        clusterList,
        ingestNames: new Set(ingestList.map(s => s.name)),
        clusterNames: new Set(clusterList.map(s => s.name)),
        podIds,
        allStreams,
    };
}
```

**Remove `withAllStreams()` method** ‚Äî no longer needed as a separate step.  
**Update `SyncOrchestratorService.execute()`** ‚Äî remove the `withAllStreams()` call.  
**Update `SyncContext` interface** ‚Äî make `allStreams` required instead of optional.
